# Tools

툴은 실제로 일을 실행하는 파이썬 파일들이야.

## 툴이 뭔가요?

신입사원이 손으로 일일이 하지 않고 계산기, 복사기, 엑셀 같은 도구를 쓰는 것과 같아. 에이전트가 머릿속으로 직접 처리하는 대신, 툴을 실행해서 결과를 받아.

## 툴이 하는 일 (예시)

- API 호출 (외부 서비스에서 데이터 가져오기)
- 데이터 수집 및 가공
- 리포트 생성
- 이메일 발송
- 파일 변환

## 툴 작성 규칙

1. **단일 책임** : 하나의 툴은 하나의 역할만 해. 여러 일을 한꺼번에 처리하게 만들지 마.
2. **명확한 입출력** : 뭘 넣으면 뭐가 나오는지 항상 예측 가능해야 해.
3. **에러 명시** : 에러가 나면 조용히 넘기지 말고, 무슨 에러인지 명확하게 알려줘.
4. **툴끼리 직접 호출 금지** : 툴이 다른 툴을 직접 부르지 마. 에이전트가 중간에서 흐름을 관리해.

## 흐름

```
에이전트가 워크플로우를 읽고 → 툴을 선택해서 실행 → 결과를 전달
```
