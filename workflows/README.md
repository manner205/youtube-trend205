# Workflows

워크플로우는 반복되는 업무의 단계별 지시서야.

## 워크플로우가 뭔가요?

첫 출근날 받는 업무 매뉴얼이라고 생각해. 에이전트(Claude)가 어떤 업무를 수행할 때, 매번 처음부터 생각하지 않도록 순서와 규칙을 미리 정리해둔 거야.

## 각 워크플로우 파일에 들어가는 내용

- **목표** : 이 워크플로우가 무엇을 달성하는지
- **입력값** : 시작하려면 무엇이 필요한지
- **단계** : 어떤 Tool을 어떤 순서로 실행하는지
- **결과물** : 완료됐을 때 무엇이 나와야 하는지
- **에러 대응** : 문제가 생기면 어떻게 하는지

## 규칙

- 한 번이라도 반복되는 업무는 워크플로우로 만들어.
- 워크플로우가 있으면 에이전트는 반드시 워크플로우대로 실행해.
- 새로운 기능이 생기거나 에러를 수정하면 관련 워크플로우도 함께 업데이트해.
- 워크플로우가 없는 새 업무는 임의로 실행하지 말고 먼저 사장한테 물어봐.
